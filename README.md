# PDP-11 CPU & Peripheral Emulator v1.0

Высокоточный эмулятор 16-битного микропроцессора архитектуры PDP-11 (LSI-11), написанный на языке C. Проект включает в себя полноценное ядро CPU, кастомный движок автоматического юнит-тестирования и эмуляцию периферийных устройств ввода-вывода (Memory-Mapped I/O).

## 🚀 Основные возможности эмулятора

*   **Полная эмуляция ОЗУ (64 Кб)**: Реализован побайтовый (`b_read`/`b_write`) и пословный (`w_read`/`w_write`) доступ к памяти с проверкой четности адресов и границ ОЗУ.
*   **Все 8 канонических мод адресации**: От регистровой (Мода 0) до индексной косвенной со знаковым расширением смещений (Мода 7).
*   **Железные правила PC и SP**: Аппаратная защита указателя стека `SP` (R6) и счетчика команд `PC` (R7) — изменение адреса для них всегда строго равно 2 байтам, даже в байтовых инструкциях.
*   **Изолированная периферия (MMIO)**: Реализована аппаратная изоляция портов дисплея. Регистр состояния `OSTAT` (0177564) всегда рапортует о готовности устройства (`0200`), а запись в регистр данных `ODATA` (0177566) перехватывается и выводится на экран терминала.
*   **Гибкий многоуровневый логгер**: Централизованная система `print_log` с поддержкой уровней логирования от глубокой отладки до полной тишины. Уровень по умолчанию — `LOG_INFO`.

## 🛠️ Поддерживаемый набор инструкций

*   **Управление процессором**: `HALT`
*   **Двухадресные**: `MOV`, `MOVB`, `ADD`
*   **Одноадресные**: `CLR`, `TSTB`
*   **Ветвления и циклы**: `SOB`, `BR`, `BNE`, `BEQ`, `BPL`
*   **Подпрограммы и стек**: `JSR` (Переход к подпрограмме), `RTS` (Возврат из подпрограммы)

## 🧪 Тестовый фреймворк

В проект встроен автономный движок автоматического тестирования, покрывающий **30 изолированных тест-кейсов**:
1.  Валидация физики чтения/записи ОЗУ и работы системного стека.
2.  Корректность декодирования опкодов и масок JSR/RTS.
3.  Проверка всех 8 режимов вычисления эффективных адресов (`get_mr`).
4.  Математика выставления флагов условий PSW (`NZVC`), включая перенос знака `C` при 32-битном сложении и выявление знакового переполнения `V`.

## 📦 Сборка и запуск проекта

Для сборки проекта вам понадобятся компилятор `gcc` и утилита `make`.

### 1. Компиляция эмулятора
```bash
make
```

### 2. Запуск автоматических тестов (Все 30 тестов)
```bash
./build/pdp11 --testall
```

### 3. Запуск одиночного теста по ID с флагом трассировки
```bash
./build/pdp11 --test 20 -trace
```

### 4. Доступные флаги логирования (управление выводом)
*   `-logoff` или `-quiet` — Полная тишина, только чистый вывод дисплея эмулятора.
*   `-error` — Выводить только критические ошибки эмуляции.
*   `-info` *(По умолчанию)* — Общая информация о работе (загрузка, приветствие, рамка останова).
*   `-trace` — Пошаговая трассировка выполнения инструкций и дамп регистров R0-R7.
*   `-debug` — Максимально глубокий лог дебага.

### 5. Пример запуска бинарного файла программы
```bash
./build/pdp11 test/10_jsr_rts/10_jsr_rts.pdp.o
```

## 📂 Структура исходного кода

*   `src/config.h` — Конфигурация размеров памяти (`MEMSIZE`) и базовых типов данных.
*   `src/cpu.h` / `src/cpu.c` — Декодер инструкций, логика адресации, выполнение команд CPU и логика стека.
*   `src/memory.h` / `src/memory.c` — Оперативная память, изоляция портов ввода-вывода дисплея (MMIO) и загрузчик бинарных файлов.
*   `src/logger.h` / `src/logger.c` — Многоуровневый системный логгер со строгой валидацией типов.
*   `src/tests.h` / `src/tests.c` — Таблица и реализация 30 автоматических юнит-тестов.
*   `src/main.c` — Главная точка входа и безопасный двухпроходный разбор параметров командной строки.
