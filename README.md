# Эмулятор процессора PDP-11 на языке C

Проект представляет собой модульный эмулятор классического 16-битного мини-компьютера PDP-11. 
Архитектура разработана с акцентом на чистоту кода, инкапсуляцию модулей и безопасность работы с битами и памятью.

## 📁 Структура проекта

```text
pdp11/
├── Makefile           # Сценарий автоматической сборки (с поддержкой ASan/UBSan)
├── .gitignore         # Фильтр исключения временных файлов и папки сборки
├── README.md          # Документация проекта
├── build/             # Изолированная папка для скомпилированного бинарника
├── test/              # Тестовые бинарные файлы программ PDP-11 (формат as11)
└── src/               # Исходный код эмулятора
    ├── config.h       # Общие типы данных (Byte, Word, Address) и макросы размеров
    ├── logger.h/.c    # Подсистема многоуровневого логирования (от LOG_OFF до LOG_DEBUG)
    ├── memory.h/.c    # Подсистема ОЗУ и загрузчик файлов
    ├── tests.h/.c     # Testing suite для отладки работы программы
    └── cpu.h/.c       # Ядро процессора: регистры R0-R7, табличный декодер и цикл run()
```

## 🛠️ Особенности реализации

* **Табличный декодер (Table-driven):** Вместо громоздких цепочек `switch-case` для разбора команд используется масштабируемый массив структур `Command` с побитовыми масками и указателями на функции.
* **Инкапсуляция памяти:** Оперативная память `mem` и регистры `reg` скрыты внутри своих модулей (`static`), доступ к ним осуществляется строго через интерфейсные функции.
* **Встроенная защита:** Сборка по умолчанию включает санитайзеры GCC (`-fsanitize=address,undefined`) для мгновенного отлова выходов за границы массивов или неопределенного поведения битовых операций.

## 🚀 Сборка и запуск

### Системные требования
Если вы используете 64-битную ОС Linux / WSL, для запуска оригинальных эталонных инструментов (ассемблера `as11`) может потребоваться поддержка 32-битной архитектуры:
```bash
sudo dpkg --add-architecture i386 && sudo apt-get update && sudo apt-get install -y libc6:i386
```

### Команды сборки
В корне проекта поддерживаются следующие правила `make`:

1. **Отладочная сборка (Рекомендуемая):**
   ```bash
   make
   ```
   *Собирает безопасную версию программы с включенными санитайзерами памяти в папку `build/pdp11`.*

2. **Быстрая сборка (Релиз):**
   ```bash
   make release
   ```
   *Отключает проверки и компилирует код с максимальной оптимизацией скорости `-O3`.*

3. **Очистка проекта:**
   ```bash
   make clean
   ```

### Запуск эмулятора
Вы можете передать эмулятору любое имя скомпилированного файла. Если файл лежит в папке `test/`, можно указывать только его короткое имя:
```bash
./build/pdp11 01_sum.pdp.o
```
